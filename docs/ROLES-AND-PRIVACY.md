# Роли, админы групп и приватность

> Согласовано с владельцем продукта. Реализация по этапам; часть полей — в миграции.

## Три уровня доступа

| Уровень | Где в БД | Кто |
|--------|----------|-----|
| **super** | `users.role = 'super'` | Владелец приложения (сейчас телефон `79680227771` / `89680227771` после нормализации). Назначает админов групп, видит всё. |
| **Админ группы** | `roster_members.is_admin = 1` | Набор людей, создание игр, голосования, оплата **в своём roster**. Создаёт аккаунты → игрок сразу в этом roster. |
| **Игрок** | `users.role = 'player'` + членство в `roster_members` | Голосует, видит состав **без чужих телефонов**. |

Глобальный `users.role = 'admin'` **устаревает** → заменяется на `super` + `is_admin` в roster.

## Создание и добавление игроков (админ группы)

1. **Новый игрок** — телефон ещё не в системе → `users` + `roster_members` (roster админа).
2. **Из списка** — телефон уже есть → только `roster_members`, **без** смены пароля.
3. Поиск: по **нику** и телефону (для админа / super).

`position` (player / goalie) — в **`roster_members`**, не глобально на `users`.

## Ник и телефон

| Поле | Назначение |
|------|------------|
| **Телефон** | Только вход до активации; админ видит при выдаче доступа. |
| **Ник** (`display_login`) | Основное имя в UI; **уникальный** при первом входе. |

## Кто видит телефон

| Кто смотрит | Видит телефон другого? |
|-------------|-------------------------|
| Сам игрок | Да, в **личном кабинете** (`/profile`) |
| Админ **этой** группы (`is_admin`) | Да, при управлении составом / добавлении |
| **super** | Да |
| Другие участники группы | **Нет** — только ник, амплуа, статусы |

В API списков состава поле `phone` **не отдаётся** обычным игрокам.

## Главная и личный кабинет (напоминание)

- **Главная:** блок игр с `vote_active` или `payment_active`; у админа группы сверху — roster, где `is_admin`.
- **Личный кабинет** (тап по иконке КХЛ): ник, **свой** телефон, команда КХЛ; список групп → состав (ники, без телефонов у чужих); админы группы — ники.
