# Go Hockey — спецификация (черновик)

Приложение для своих: один город, доступ через администратора группы.

Подробно: [ROSTERS-AND-VOTING.md](./ROSTERS-AND-VOTING.md), [ROLES-AND-PRIVACY.md](./ROLES-AND-PRIVACY.md).

## Роли

| Роль | Описание |
|------|----------|
| **super** | Владелец: назначение админов групп, полный обзор. Сейчас: учётка `79680227771` (в seed — `admin.role: "super"`). |
| **Админ группы** | `roster_members.is_admin` — создание игроков в своём roster, игры, голосования, оплата. |
| **player** | Участник; в UI везде **ник**, не телефон. |

## Roster и игры

- **Roster** — постоянный пул («Среда · ЛД Кристалл», позже другие).
- **Игра** — дата + голосование / оплата.
- Состав: **20 полевых** + резерв + **2 вратаря** (`position` в `roster_members`).

## Приватность телефона

- Телефон видит **только сам игрок** (личный кабинет) и **админ группы** / **super**.
- Участники группы в списках видят **только ник** (и амплуа).

## Поток пользователя

### 1. Выдача доступа (админ группы)

1. Админ группы: **телефон** + временный пароль → аккаунт сразу в **его** roster.
2. Добавление в другую группу: **поиск существующего** по нику/телефону, без второго аккаунта.
3. Данные для входа админ отправляет **вне** приложения.

### 2. Первый вход (игрок)

1. Вход: телефон + временный пароль.
2. Команда КХЛ, **ник** (уникальный), новый пароль.
3. Аккаунт активен; членство в roster уже есть.

### 3. Голосование

Админ группы запускает голосование с подписями ответов → push пулу → состав 20 + резерв + 2 GK.

### 4–7. Состав, оплата, гости

Как в [ROSTERS-AND-VOTING.md](./ROSTERS-AND-VOTING.md).

## Главная (целевой UI)

| Пользователь | Блоки |
|--------------|--------|
| Админ группы (`is_admin`) | «Мои группы» (управление) + «Сейчас важно» (игры с голосованием/оплатой в roster, где состоит) |
| Игрок | Только «Сейчас важно» (те же игры, только свои roster) |

Личный кабинет: тап по иконке КХЛ слева.
