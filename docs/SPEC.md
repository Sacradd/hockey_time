# Go Hockey — спецификация (черновик)

Приложение для своих: один город, доступ через администратора группы.

Подробно: [ROSTERS-AND-VOTING.md](./ROSTERS-AND-VOTING.md), [ROLES-AND-PRIVACY.md](./ROLES-AND-PRIVACY.md), [POOL-AND-GROUPS.md](./POOL-AND-GROUPS.md), [DEV-SEQUENCE.md](./DEV-SEQUENCE.md).

## Роли

| Роль | Описание |
|------|----------|
| **super** | Владелец: назначение админов групп, полный обзор. Локально: `admin` / `admin` (seed `display_login` + тел.-заглушка `79000000001`). |
| **Админ группы** | `users.role = admin` (создатель групп, без привязки к льду) или `roster_members.is_admin` — своя группа, игроки, игры, голосование. |
| **player** | Участник; в UI везде **ник**, не телефон. |

**Пул и группы:** все аккаунты в общем списке (`users`); связь с группой — `roster_members`. Super добавляет в пул; админ создаёт группу и привязывает игроков. См. [POOL-AND-GROUPS.md](./POOL-AND-GROUPS.md).

## Roster и игры

- **Roster** — постоянный пул («Среда · ЛД Кристалл», позже другие).
- **Игра** — дата + голосование / оплата.
- Состав: **20 полевых** + резерв + **2 вратаря** (`position` в `roster_members`).

## Приватность телефона

- Телефон видит **только сам игрок** (личный кабинет) и **админ группы** / **super**.
- Участники группы в списках видят **только ник** (и амплуа).

## Поток пользователя

### 1. Выдача доступа

1. **Super:** телефон + пароль в **общий пул**; опционально «будущий админ групп» (`role = admin`, без группы).
2. **Админ:** создаёт **группу** (обязательно только **название**; площадка/день — при игре); в группе **+** — новый в пул и связь, или поиск по пулу.
3. Данные для входа передаются **вне** приложения.

### 2. Первый вход (игрок)

1. Вход: телефон + временный пароль.
2. Команда КХЛ, **ник** (уникальный), новый пароль.
3. Аккаунт активен; членство в roster уже есть.

### 3. Голосование

Админ группы запускает голосование с подписями ответов → push пулу → состав 20 + резерв + 2 GK.

### 4–7. Состав, оплата, гости

Как в [ROSTERS-AND-VOTING.md](./ROSTERS-AND-VOTING.md).

## Главная (целевой UI)

| Пользователь | Блоки |
|--------------|--------|
| Админ группы (`is_admin`) | **Группы** (карточки, «Кол-во участников») + **Игры** (предстоящие по дате и/или с активным голосованием/оплатой; пусто: «Нет предстоящих игр») |
| Игрок | Только **Игры** (те же игры, только свои roster) |

Карточка игры на главной: **сверху** — название/дата игры; **снизу** — «Группа — {название группы}» (площадка через ` · `). Табличка `.roster-name-plate` — только на странице управления группой (`RosterPage`), не на главной. Редактирование/удаление **группы** — карандаш в шапке roster (`update-roster.php`); **игры** — карандаш на экране игры (`update-game.php`, `delete-game.php`).

Личный кабинет: тап по иконке КХЛ слева.
