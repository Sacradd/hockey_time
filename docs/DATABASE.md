# База данных: первая группа и пользователи

## Что создаётся

| Сущность | Описание |
|----------|----------|
| **admin** | Один администратор (телефон + пароль), сразу активен |
| **players** | Игроки из списка в `seed.json`, временный пароль, `must_change_password = 1` |
| **day_groups** | Первая группа по дате (игровой день) |
| **group_members** | Все привязаны к группе; у админа `actual = 1`, у игроков `actual = 0` до первого входа |

После **этапа 1 (авторизация)** при первом входе игрок станет `is_active = 1` и в группе `actual = 1`.

**Сначала разработка локально:** [LOCAL.md](./LOCAL.md) (Docker). reg.ru — когда схема и вход протестированы.

## Шаги на reg.ru

### 1. MySQL в панели

1. Создайте базу и пользователя MySQL.
2. Запомните: хост, имя БД, логин, пароль.

### 2. Файлы на хостинг

```
public_html/          ← dist/ (PWA)
public_html/api/      ← папка api/ из репозитория
database/             ← рядом с api или вне public_html (seed.json не в git)
```

Рекомендация: `database/seed.json` положить **вне** `public_html`, если панель позволяет; иначе закройте доступ `.htaccess`.

### 3. Конфиг

```bash
cp api/config.example.php api/config.local.php
```

Заполните `db` и `install_secret`.

### 4. Список людей

```bash
cp database/seed.example.json database/seed.json
```

Отредактируйте:

- `admin.phone` — ваш телефон (только цифры, можно `8...` или `7...`)
- `admin.password` — пароль для входа админа
- `group.date` — дата первой игры (`YYYY-MM-DD`)
- `group.title` — подпись (необязательно)
- `players` — массив `{ phone, password }` для каждого игрока

`database/seed.json` **не коммитится** (в `.gitignore`).

### 5. Установка

Откройте в браузере (подставьте домен и secret):

```
https://ваш-домен.ru/api/install.php?secret=ВАШ_install_secret
```

Ответ `{"ok":true,...}` — таблицы и данные созданы.

**Удалите `api/install.php`** с сервера после установки.

## Повторный запуск

Скрипт идемпотентен: существующие телефоны и дата группы обновляются, дубликаты не создаются.
