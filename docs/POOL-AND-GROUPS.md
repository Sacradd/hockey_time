# Общий пул и группы (согласовано)

## Сущности

```
users                    — все учётные записи (общий пул)
rosters                  — группа (площадка, день недели, название)
roster_members           — связь user ↔ roster, is_admin, position
day_groups / votes / …   — игры внутри группы
```

## Super

| Действие | Где в UI | API |
|----------|----------|-----|
| Создать игрока в **пул** (без группы) | Главная → Пользователи → **+** | `POST /api/admin/create-user.php` |
| При создании: «Будет админом групп» | Чекбокс на форме | `is_group_admin` → `users.role = admin` |
| Сброс пароля, удаление, админ **в группе** | Тап по игроку в списке | `reset-password`, `delete-user`, `set-roster-admin` |
| Создавать группы | **Нет** | — |

Super **не** привязывает игрока к группе при создании — админ сам создаст группу и добавит людей.

## Админ группы (`users.role = admin` или `roster_members.is_admin = 1`)

| Действие | Где в UI | API |
|----------|----------|-----|
| Создать **группу** | Главная → **+** (оранжевый круг) | `POST /api/admin/create-roster.php` |
| Войти в группу | Управление группами | — |
| Добавить игрока | **+** в группе | Новый: `create-user` + link или `create-player` с `roster_id` |
| Добавить из пула | «Из списка» в группе | `add-member.php` |
| Убрать из группы | Свайп влево по строке | `remove-member.php` |
| Назначать других админов | **Нет** (только super) | — |

## Два пути добавления в группу

1. **Новый человек** — запись в `users` + `roster_members` для текущей группы.
2. **Уже в пуле** (создан super без группы) — только `roster_members`.

## Отвязка vs удаление

| Операция | Кто | Эффект |
|----------|-----|--------|
| Отвязка от группы | Админ группы | Удаляется `roster_members`, `users` остаётся |
| Удаление из приложения | Super | `DELETE users` (каскад по FK) |
